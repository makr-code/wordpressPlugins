# WordPress Shop Hook Overrides

This guide shows how to override recommendation behavior for the dynamic shop shortcode in the ThemisDB Order Request plugin.

## Goal

Customize which edition is recommended for:
- Main shop CTA
- Module deep links
- Training deep links

without changing plugin core files.

## Prerequisites

- WordPress with ThemisDB Order Request plugin active
- Access to filesystem
- Permission to add MU plugins

## Install

1. Copy the example file from:
   wordpress-integration-example/mu-plugins/themisdb-shop-recommendation-overrides.php

2. Place it in your WordPress instance at:
   wp-content/mu-plugins/themisdb-shop-recommendation-overrides.php

3. Open the Shop page and verify links generated by [themisdb_shop].

## Available hooks

### 1) Preference order hook

Hook name:
- themisdb_shop_recommended_edition_preferences

Use this to return an ordered array of editions for the current context.

Input parameters:
- preferences (array)
- context (default|module|training)
- detail (module category or training type)
- available editions
- products snapshot
- license price snapshot

### 2) Final edition hook

Hook name:
- themisdb_shop_recommended_edition

Use this when you want to force a final edition for selected cases.

Input parameters:
- resolved edition
- context
- detail
- active preference list
- available editions
- products snapshot
- license price snapshot

## Validate quickly

Run smoke script:

pwsh -File scripts/themisdb-order-request-e2e-smoke.ps1 -WpPath "C:\path\to\wordpress"

Expected relevant outputs include:
- Dynamic shop shortcode registered
- Dynamic shop shortcode renders base markup
- Order flow applies edition/module/training checkout presets

## Notes

- MU plugins load automatically on every request.
- Keep conditions explicit and deterministic.
- Prefer hook-based customization over editing plugin source directly.
